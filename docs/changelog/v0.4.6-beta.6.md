# v0.4.6-beta.6

- Recover from stale `apply.lock` directories after interrupted plugin runs instead of leaving the state dir permanently blocked.
- Report state-dir permission failures as explicit filesystem errors instead of misleading lock-conflict errors.
- Degrade session file logging gracefully when `ENVLOCK_LOG_HOME` is unwritable.
- Route npm and yarn global installs into plugin-controlled prefixes so installed binaries are executable through the selected toolchain path.
